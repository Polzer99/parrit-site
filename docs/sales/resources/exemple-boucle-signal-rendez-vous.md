# Ressource : boucle signal -> qualification -> message -> rendez-vous

Date : 2026-06-24
Statut : source envoyable / a designer en PDF
Usage : directeur commercial, CCO, CEO oriente croissance, CMO demand generation

## Intention

Cette ressource montre comment Parrit transforme une prospection froide en boucle de signaux qualifies. Elle doit rassurer sur un point : on ne cherche pas a envoyer plus, on cherche a contacter mieux, avec un humain qui garde la validation.

Elle est utile quand le prospect dit :

- "Nos commerciaux partent de listes trop froides."
- "On manque de contexte pour personnaliser."
- "La prospection prend trop de temps pour trop peu de rendez-vous."
- "Je ne veux pas industrialiser du bruit."

## Version courte envoyable

### La boucle commerciale Parrit

La prospection devient performante quand elle part d'un moment utile.

Un signal peut etre une actualite, un recrutement, un changement d'organisation, une levee, une ouverture de site, une prise de parole, un lancement produit ou une contrainte reglementaire.

La boucle Parrit transforme ce signal en action commerciale controlee :

`Signal detecte` -> `compte qualifie` -> `angle prepare` -> `message propose` -> `validation humaine` -> `suivi de campagne`

## Les 5 etapes

| Etape | Ce qui se passe | Sortie attendue |
|---|---|---|
| 1. Detection | La machine surveille des sources utiles au secteur. | Une liste courte de signaux datables. |
| 2. Qualification | Le compte est verifie : taille, secteur, pertinence, priorite. | Seuls les comptes actionnables restent. |
| 3. Angle | Le signal est traduit en raison de contact. | Une hypothese commerciale claire. |
| 4. Message | Un premier message est prepare dans la voix de l'entreprise. | Une prise de contact prete a valider. |
| 5. Pilotage | Reponses, non-reponses et exceptions sont suivies. | Un tableau lisible pour ajuster la boucle. |

## Avant / apres

| Avant | Apres |
|---|---|
| Listes achetees ou exportees sans contexte. | Comptes attaches a un signal recent et verifiable. |
| Messages generiques, faibles reponses. | Angle personnalise, relance coherente, validation humaine. |
| Les commerciaux cherchent eux-memes l'information. | Le contexte arrive prepare, avec la source et le pourquoi. |
| Le suivi depend de routines individuelles. | Reponses, relances et exceptions sont pilotees dans une file claire. |

## Exemple anonymise

Situation : une equipe commerciale B2B veut identifier des comptes qui viennent de vivre un evenement rendant l'offre pertinente.

Flux cible :

1. Surveiller les sources publiques du secteur.
2. Extraire l'entreprise, le signal, la date et le contexte.
3. Verifier taille, secteur, zone et potentiel.
4. Proposer un angle de message.
5. Laisser l'humain valider avant l'envoi.
6. Suivre la reponse et enrichir la prochaine action.

Ce que le prospect voit : moins de bruit, plus de contexte, une file de comptes vraiment travaillables.

## Use cases

- Courtage energie : signal marche, changement de contrat, ouverture de site, croissance multi-sites.
- SaaS B2B : recrutement d'une fonction cle, migration outil, annonce client, levee de fonds.
- Conseil : nouveau dirigeant, reorganisation, annonce de plan strategique.
- E-commerce B2B : lancement catalogue, extension geographique, nouveau canal de vente.
