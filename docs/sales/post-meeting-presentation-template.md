# Template prez post-meeting par persona

Date : 2026-06-24
Statut : template interne
Usage : support personnalisable apres rendez-vous
Sources : `docs/sales/persona-value-prop-matrix.md`, `docs/sales/post-meeting-template.md`, `MATURITE-SOT.md`

Objectif : produire vite une suite de rendez-vous qui parle a la personne rencontree. Le support doit raconter son quotidien, montrer un avant / apres concret, proposer deux ou trois cas d'usage et terminer par un prochain pas simple.

Ce template n'est pas une propale finale. Il devient une page protegee ou une presentation partageable une fois personnalise avec le nom, les cas, les exemples et les ressources utiles.

## Regle de construction

1. Choisir le persona principal et, si besoin, un persona secondaire.
2. Garder 6 a 8 slides maximum.
3. Placer l'avant / apres avant la description Parrit.
4. Montrer un cas prioritaire et deux options, pas un catalogue.
5. Adapter la ressource envoyee au niveau de maturite N1 a N7.
6. Garder les preuves anonymisees tant que le support n'est pas protege.

## Structure de la prez

### Slide 1 - Ce qu'on a compris

Titre : `{prenom}, le sujet n'est pas "faire de l'IA". C'est {enjeu_persona}.`

Blocs :

- Moment : `{moment}`
- Friction visible : `{douleur_1}`
- Risque a proteger : `{risque}`
- Decision utile : choisir un premier flux qui peut tourner, etre mesure, puis etre etendu.

### Slide 2 - Votre realite aujourd'hui

Format : trois cartes courtes.

| Carte | Contenu |
|---|---|
| Charge | Ce qui revient trop souvent et consomme l'equipe. |
| Dispersion | Ou l'information se perd entre personnes, outils et canaux. |
| Controle | Ce qui doit rester humain, trace et gouverne. |

### Slide 3 - Avant / apres

| Avant | Apres |
|---|---|
| `{avant_1}` | `{apres_1}` |
| `{avant_2}` | `{apres_2}` |
| `{avant_3}` | `{apres_3}` |

Phrase a garder :

> La valeur n'est pas dans l'effet de demonstration. Elle apparait quand un flux important devient plus clair, plus rapide et plus controlable.

### Slide 4 - Trois cas possibles, un seul a choisir

| Cas | Pourquoi lui | Ce qu'on verrait vite |
|---|---|---|
| `{use_case_prioritaire}` | `{raison_priorite}` | `{signal_resultat}` |
| `{use_case_option_2}` | `{raison_option_2}` | `{signal_option_2}` |
| `{use_case_option_3}` | `{raison_option_3}` | `{signal_option_3}` |

### Slide 5 - Le premier flux cible

Schema texte :

`Entrees` -> `Tri / enrichissement` -> `Action preparee` -> `Validation humaine` -> `Trace + mesure`

Details a completer :

- Entrees : `{canaux_entree}`
- Outil de destination : `{outil_destination}`
- Exceptions : `{exceptions}`
- Validation : `{validateur}`
- Mesure : `{kpi}`

### Slide 6 - Comment Parrit intervient

| Role | Ce que ca apporte |
|---|---|
| Cadrage metier | Choisir le bon cas, le bon perimetre et le bon niveau de risque. |
| Construction | Produire un flux utilisable sur les outils existants autant que possible. |
| Mise sous controle | Garder validation humaine, traces, indicateurs et transfert equipe. |
| Extension | Decider quoi etendre apres preuve du premier flux. |

### Slide 7 - Ressource jointe

Choisir une ressource, pas trois.

| Situation | Ressource |
|---|---|
| N1 / decouverte | Carte "De l'IA generative a l'IA agentique" avec 7 niveaux. |
| Douleur operationnelle | Mini-schema avant / apres du flux prioritaire. |
| Sponsor dirigeant | Note 1 page : deux fronts, premier chantier, risque traite. |
| Grand groupe / DSI | Note gouvernance : agnostique, donnee chez vous, coeur intact. |
| Commercial / marketing | Exemple boucle signal -> qualification -> message -> rendez-vous. |
| E-commerce | Exemple scan visibilite achat IA ou back-office client multi-canal. |
| Deal chaud | Page propale privee avec prix, destinataires et tracking. |

### Slide 8 - Prochain pas

CTA a personnaliser :

> On prend un exemple reel de `{use_case_prioritaire}` et on cartographie le flux cible en 45 minutes : entrees, exceptions, validation, mesure et premiere version possible.

## Propositions de valeur par personne

### DG / CEO

Enjeu : passer de l'idee IA au premier chantier qui rend du temps et prouve la methode.

Avant / apres :

| Avant | Apres |
|---|---|
| Initiatives dispersees, personne ne sait par ou commencer. | Un flux prioritaire choisi, opere et mesure. |
| Les equipes testent mais rien ne devient une capacite interne. | Une methode replicable pour les prochains chantiers. |

Use cases :

- back-office multi-canal ;
- qualification commerciale par signal ;
- reporting dirigeant ;
- veille marche.

Ressource : carte des 7 niveaux + note "deux fronts".

CTA : "On choisit le premier chantier qui doit tourner en quelques semaines."

### Owner e-commerce

Enjeu : sortir le fondateur des taches repetitives client sans casser la relation de marque.

Avant / apres :

| Avant | Apres |
|---|---|
| Messages, commandes et leads eparpilles dans plusieurs canaux. | Base unique, segments propres, relances suivies. |
| SAV et relances au feeling. | Reponses preparees, exceptions remontees, humain sur la decision. |

Use cases :

- capture TikTok / Snap / site / telephone ;
- relances WhatsApp segmentees ;
- tri SAV ;
- newsletter de marque.

Ressource : schema "canaux -> base unique -> validation humaine".

CTA : "On prend un flux client et on le rend operable."

### Directeur commercial / CCO

Enjeu : remplacer la prospection froide par une boucle de signaux qualifies.

Avant / apres :

| Avant | Apres |
|---|---|
| Listes froides, messages generiques, pipe irregulier. | Signaux detectes, messages personnalises, validation humaine. |
| Les commerciaux passent du temps a chercher le contexte. | Le contexte arrive prepare avec le compte et le moment utile. |

Use cases :

- veille comptes cibles ;
- enrichissement ;
- sequences personnalisees ;
- relance non-reponses ;
- tableau de pilotage.

Ressource : exemple "signal -> qualification -> message -> rendez-vous".

CTA : "On prend un signal reel de votre secteur et on teste la boucle de bout en bout."

### Directeur e-commerce retail

Enjeu : mesurer la visibilite de la marque dans les reponses d'achat IA et corriger ce qui fait perdre la citation.

Avant / apres :

| Avant | Apres |
|---|---|
| Aucune lecture des questions ou la marque disparait. | Taux de citation par segment et actions exactes sur catalogue. |
| Service client et contenus produit traites separement. | Scan, correction, re-mesure et back-office plus leger. |

Use cases :

- scan Rufus par segment ;
- correction fiches produit ;
- tri service client ;
- relances multi-canal.

Ressource : exemple "question acheteur -> citation -> correction -> re-mesure".

CTA : "On lance un premier scan sur un segment de catalogue."

### Directeur transformation / innovation

Enjeu : passer de chantiers disperses a une pratique agentique gouvernee et portee par les equipes internes.

Avant / apres :

| Avant | Apres |
|---|---|
| Beaucoup d'initiatives locales, peu de passage en production. | Un cas metier en production, referents formes, playbook de replication. |
| Dependances externes et gouvernance floue. | Revue d'alignement, garde-fous et capacite interne. |

Use cases :

- pilote metier ;
- formation referents ;
- revue d'alignement ;
- cadre de gouvernance.

Ressource : note "pilote en production + enablement en cascade".

CTA : "On choisit un cas prioritaire et le cadre de gouvernance autour."

### COO

Enjeu : enlever le geste repetitif qui ralentit les operations et rendre le process mesurable.

Avant / apres :

| Avant | Apres |
|---|---|
| Boites mail partagees, escalades, erreurs de ressaisie. | Tri, preparation, escalade ciblee et action dans l'outil. |
| La charge reelle est invisible. | Volumes, delais, exceptions et actions sont suivis. |

Use cases :

- reclamations ;
- commandes ;
- alertes rupture ;
- synchronisation ERP / tableur.

Ressource : schema avant / apres du flux operationnel.

CTA : "On part du geste quotidien le plus couteux et on le transforme en file supervisee."

### DAF / CFO

Enjeu : gagner du temps sur les flux financiers repetitifs sans perdre la trace, le controle et la defense du ROI.

Avant / apres :

| Avant | Apres |
|---|---|
| Cloture longue, rapprochements manuels, reporting fragile. | Base consolidee, relances, trace et validation humaine. |
| Le gain est difficile a defendre. | Chaque etape montre volume, temps rendu et risque traite. |

Use cases :

- rapprochement ;
- relances clients ;
- reporting ;
- controle processus ERP.

Ressource : mini-audit 1 page avec risques et indicateurs.

CTA : "On choisit un flux financier repetitif et on definit la piste d'audit attendue."

### DRH

Enjeu : faire adopter l'IA par des outils utiles le lundi matin, pas par une formation abstraite.

Avant / apres :

| Avant | Apres |
|---|---|
| Formation generique, oubli rapide, inquietude equipe. | Cas RH reel, referents internes et outil reutilisable. |
| Back-office RH absorbe par les memes demandes. | Tri, premiere reponse et escalade humaine. |

Use cases :

- tri CV ;
- reponses candidats ;
- onboarding ;
- FAQ paie ;
- cascade referents.

Ressource : trame atelier referents + exemple de cas construit en live.

CTA : "On choisit un cas RH concret et on forme les referents autour."

### DSI / CIO

Enjeu : capter la valeur agentique sans exposer le coeur du SI, les donnees ou la gouvernance.

Avant / apres :

| Avant | Apres |
|---|---|
| Usages disperses, shadow IT, risque data. | Environnements separes, acces controles, journaux d'audit. |
| Les metiers veulent aller vite, le SI porte le risque. | Les agents orchestrent les satellites sans toucher au coeur. |

Use cases :

- agents sur outils satellites ;
- controle acces ;
- journaux d'audit ;
- transfert aux equipes.

Ressource : note architecture agnostique.

CTA : "On cartographie le flux satellite, les donnees et les garde-fous avant toute construction."

### CMO

Enjeu : transformer l'expertise de marque en contenus et signaux mesurables sans diluer la voix.

Avant / apres :

| Avant | Apres |
|---|---|
| Calendrier tendu, dependance agence, faible reutilisation. | Content factory, voix preservee, actifs declinables. |
| Visibilite IA non mesuree. | Citations suivies, corrections ciblees, re-mesure. |

Use cases :

- articles ;
- posts ;
- videos ;
- repurposing ;
- mesure citations ;
- localisation.

Ressource : exemple content factory + mesure GEO.

CTA : "On part d'un actif existant et on construit la boucle contenu + mesure."

### Directeur juridique

Enjeu : accelerer la premiere passe et la veille sans deleguer la decision juridique.

Avant / apres :

| Avant | Apres |
|---|---|
| Contrats relus de zero, veille chronophage. | Clauses marquees, risques tries, veille supervisee. |
| La qualite depend de la memoire individuelle. | Methode, trace et validation par le juriste. |

Use cases :

- revue contrats ;
- matrice de risque ;
- veille reglementaire ;
- pre-annotation.

Ressource : note "humain dernier filtre" + exemple veille supervisee.

CTA : "On prend un type de document ou une source de veille et on definit la premiere passe utile."

### Avocat associe

Enjeu : proteger le temps facturable et le secret professionnel tout en accelerant la preparation.

Avant / apres :

| Avant | Apres |
|---|---|
| Veille et mise en forme mangent le temps facturable. | Signaux qualifies, premieres passes et validation avocat. |
| Les sources restent a surveiller manuellement. | File de veille controlee, citations et trace. |

Use cases :

- veille portefeuille clients ;
- sources officielles ;
- structuration d'actes ;
- supervision interne.

Ressource : exemple de veille sur sources officielles.

CTA : "On choisit une veille ou une premiere passe qui reste sous validation avocat."

### Expert-comptable associe

Enjeu : transformer la chasse aux pieces en file de validation pour rendre du temps au conseil.

Avant / apres :

| Avant | Apres |
|---|---|
| Pieces en vrac, collaborateurs absorbes par la saisie. | Classement, manquants detectes, pre-rapprochement. |
| Relances et echeances reposent sur la vigilance humaine. | File claire, escalades et trace par dossier. |

Use cases :

- collecte multi-canal ;
- GED ;
- rapprochement bancaire ;
- veille conseil client.

Ressource : schema collecte -> manquants -> validation.

CTA : "On prend un dossier type et on cartographie la file de validation."

| `{prenom}` | Prenom de la personne rencontree. |

## Checklist qualite

- La premiere slide parle de la personne, pas de Parrit.
- Il y a un avant / apres concret avant les moyens.
- Le support propose un premier flux, pas une transformation totale.
- La ressource jointe correspond au niveau de maturite.
- Les preuves restent anonymisees si le support n'est pas protege.
- Le CTA est une action de cadrage simple et dateable.
